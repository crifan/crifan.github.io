又报同样错误。

错误原因：Playwright的内部实现机制，个人感觉，虽然看起来很高级，但是实际上用起来很是麻烦。导致，如果你是多次循环类的调用，或者是Python中的函数作为参数等情况，去初始化browser时，就会被内部判断为，视为，异步调用。

解决办法：只能把代码，改为，整套代码运行期间，全局只能初始化一次browser。

否则如果有多次初始化调用sync_playwright().start()就会报错。

比如我此处最后的代码是：
