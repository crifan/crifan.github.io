XinaA15越狱

用XinaA15越狱自动实现可调试

  • XinaA15越狱后：自带app可调试
    • 优点
      • XinaA15越狱期间会自动给debugserver重签名，加了可调试权限，使得后续全局范围内，可以调试任何app
    • 缺点
      • 只适用于：A12+芯片的iPhone机型
        • 注：XinaA15只支持A12+的机型

XinaA15越狱期间

  • XinaA15开启越狱期间
    • 现象1
      • Signature Debugserver=重签名Debugserver
    • 现象2
      • 签名debugserver
  • 估计就是指的是给debugserver重签名
    • 使得其支持：所有进程可调试
    • 得到我们要的效果：XinaA15越狱后，任何进程均可调试

XinaA15越狱后

越狱后的debugserver

  • 进程管理器中的debugserver的entitlement权限中，就有了：task_for_pid-allow
  • -> 给debugserver加上可调试权限 -》 可以调试任意app/进程了

细节是：

XinaA15中的debugserver的进程：

  • debugserver：显示已被(注入)(P)
  • 点击弹框可以查看到
    • 已注入了库：xxx/procursus/usr/lib/TweakInject.dylib
  • 去查看ENT=entitlement=权限，发现已经有我们希望的值了
    • task_for_pid-allow
    • get-task-allow

越狱后的其他app或二进制

设置=Preferences

iOS内置app：设置=Preferences

  • 已注入
      • 进程PID是2354

即可顺利调试Preferences：

akd

XinaA15中的akd的进程：

  • 去查看ENT=entitlement=权限，发现已经有我们希望的值了
    • task_for_pid-allow + proc_info-allow
    • get-task-allow
