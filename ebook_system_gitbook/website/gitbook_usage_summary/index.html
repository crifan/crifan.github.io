调试时添加--log以便于看到详细的错误信息

在用serve去调试本地的gitbook时，有些细节和错误，比如markdown的插件中语法写错了，是不知道，也看不到错误信息的。

而想要看到，则需要添加--log的参数，可以输出详细的log日志信息，从而可以看出原因和细节。

比如，语法高亮插件中用的是prism，而markdown源码写了个md的语法：

```markdown
`Markup` : `markup`
```

时，结果log中即可看到错误信息：
